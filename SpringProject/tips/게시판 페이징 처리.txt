페이징 처리

* 교육과정에선 간단한 예제만 하므로 게시판 리스트를 불러올 때 select * from tbl_board; 와 같이 전체 조회 쿼리를 날리지만
실무에서는 데이터가 수십만, 수백만 건이 넘어가므로 과부하 문제가 발생할 수 있다. 이를 방지하기 위해 다음과 같이 처리한다.

* 오라클 튜닝 / 데이터 튜닝 -> 데이터베이스 조회 성능 올리는 작업 (속도향상, 부하줄이기 등)

1. 최소한의 데이터를 전송 (서버)
2. 필요한 것만 보는 것

url에 문자열을 조절해서 원하는 페이지 호출
페이지 하단에 페이지 번호를 보여주고 클릭하여 이동
페이지에서 조회나 수정 작업 후에 다시 원래의 페이지로 이동

GET 방식

1페이지에 10개의 글을 게시 - 32건의 데이터가 있을시 4개 페이지 필요

필요한 쿼리문
1. 페이징 처리 SQL
2. 데이터 개수를 세는 SQL (count(*))
자바스크립트 또는 a태그를 통한 이벤트 처리

Mysql limit 기능
10개씩 데이터를 출력할 경우
select ...
where ...
order by ...
limit 시작데이터, 데이터 개수

ex) 10개씩 데이터를 출력하는 경우
	1page limit 0, 10
	2page limit 10, 10
	
	20개씩 데이터를 출력하는 경우
	1page limit 0, 20
	2page limit 20, 20
